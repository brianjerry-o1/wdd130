Task 3: Nesting Block in Inline

Try placing a <div> inside a <span> element.
This is a block-level div inside an inline span.

What error or layout issue do you see? Why does this happen?

Answer: The block-level div forces itself onto a new line, "breaking" the span container. This happens because browsers expect inline elements to contain only data or other inline elements, not structural blocks.
